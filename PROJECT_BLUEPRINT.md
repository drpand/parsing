## 🏗️ 2. АРХИТЕКТУРА (LOGIC-FIRST)

### Принципы (НЕПРЕРЕКАЕМЫЕ)

```python
# ✅ ДЕЛАЕМ:
1. Модульность: каждый компонент — независимый сервис
2. Асинхронность: IO-операции не блокируют event loop
3. Идемпотентность: повторный запрос = тот же результат (где возможно)
4. Конфигурируемость: все "магические числа" — в config/env
5. Наблюдаемость: каждое действие — лог с контекстом

# ❌ НЕ ДЕЛАЕМ:
1. Хардкод секретов и URL
2. Синхронные вызовы в асинхронном контексте
3. Прямые изменения в рабочих модулях без тестирования
```

### Схема компонентов (логическая)

```
┌─────────────────────────────────────────┐
│           TELEGRAM BOT (aiogram)         │
│  • Обработка команд                      │
│  • Управление состояниями (FSM)          │
│  • Отправка медиа/кнопок                 │
└─────────────┬───────────────────────────┘
              │ (события)
              ▼
┌─────────────────────────────────────────┐
│           CORE ORCHESTRATOR             │
│  • dispatcher.py                         │
│  • Роутеры (9 шт)                        │
│  • Middleware                            │
└─────────────┬───────────────────────────┘
              │
    ┌─────────┼─────────┐
    ▼         ▼         ▼
┌────────┐ ┌────────┐ ┌──────────┐
│ Parser │ │  AI    │ │ Database │
│Service │ │Service │ │ Service  │
└────────┘ └────────┘ └──────────┘
```

**🔑 Важно:** Логика связей между компонентами важнее, чем конкретная реализация. Можно заменить Selenium на Playwright, OpenRouter на свою модель — но поток данных должен остаться таким же.

## 🔄 4. ПОТОК ДАННЫХ (DATA FLOW)

### Сценарий: Пользователь запрашивает товар

```
User → /find 123 → Bot → Database → ProductDTO → Formatter → Message → User
```

### Ключевые DTO (Data Transfer Objects)

```python
@dataclass
class ProductDTO:
    url: str
    title: str
    price_inr: float
    price_rub: float  # Конвертированный
    images: List[str]
    description: str  # От ИИ
    attributes: Dict[str, Any]  # Размер, цвет, материал
    is_in_stock: bool
```

**📌 Правило:** Все модули общаются через DTO, а не "голые" словари. Это сохраняет контракт при рефакторинге.

```
        /🔺\       E2E тесты (10%) — полный сценарий пользователя
       /____\      Интеграционные (20%) — взаимодействие модулей

```

### Минимальный набор для старта

```python
# tests/unit/test_parser.py
async def test_parse_price_variations():
    """Парсер должен корректно извлекать цены в разных форматах"""
    cases = [
        ("₹1,299", 1299),
        ("1.299 INR", 1299),
        ("Rs. 1299.00", 1299),
    ]
    for text, expected in cases:
        assert extract_price(text) == expected
```

**🎯 Правило:** Тесты должны проверять поведение, а не реализацию. Если поменяли Selenium на Playwright — тесты не должны ломаться.

## 🆘 9. ТРАБЛШУТИНГ (ЧАСТЫЕ ПРОБЛЕМЫ)

| Симптом | Возможная причина | Решение |
|---------|-------------------|---------|
| Бот не отвечает | Event loop заблокирован | Проверить синхронные вызовы |
| Парсинг возвращает пустоту | Сайт изменил структуру | Проверить селекторы CSS |
| Myntra: нет изображений | Фильтр домена | Проверить `myntassets.com` в фильтрах |
| OpenRouter таймаутит | Большой промпт | Уменьшить контекст, добавить retry |
| БД "locked" | SQLite + конкурентная запись | Включить WAL-режим |
| Утечка памяти | Не закрытые браузеры | Проверить `async with` |
