# Суперспособность: Systematic Debugging (Системная отладка)

## Назначение
Методичный поиск и исправление ошибок с использованием научного подхода.

## Когда использовать
- Код не работает как ожидается
- Тесты падают с ошибкой
- Необычное поведение программы
- Перед тем как гуглить ошибку

## Алгоритм debugging

### Шаг 1: Воспроизведение
```
1. Описать ожидаемое поведение
2. Описать фактическое поведение
3. Найти минимальный сценарий воспроизведения
4. Убедиться что ошибка стабильная
```

### Шаг 2: Сбор информации
```
1. Прочитать полный текст ошибки (не пропускать!)
2. Найти где возникает (файл, строка, функция)
3. Проверить логи (bot.log, консоль)
4. Выяснить при каких условиях появляется
```

### Шаг 3: Гипотезы
```
Минимум 3 гипотезы почему ошибка:
- Гипотеза А: Проблема в данных (входные значения)
- Гипотеза Б: Проблема в логике (алгоритм)
- Гипотеза В: Проблема в окружении (версии, пути)
```

### Шаг 4: Проверка гипотез
Для каждой гипотезы:
1. Добавить логирование/принты
2. Запустить сценарий воспроизведения
3. Подтвердить или опровергнуть

### Шаг 5: Исправление
```
1. Найти корневую причину
2. Предложить исправление
3. Исправить → Тестировать
4. Убедиться что не сломало другое
```

## Пример отладки

```markdown
## 🐛 Debugging: Бот не отвечает на /start

### Воспроизведение
**Ожидание:** Бот отправляет главное меню с кнопками  
**Фактика:** Бот молчит, в логах ничего

**Сценарий:**
1. Запустить бота
2. Отправить /start в Telegram
3. Нет ответа

### Текст ошибки
```
Ошибок нет, но в логах:
2026-03-23 10:15:00 INFO: Bot started
2026-03-23 10:15:05 INFO: Update received from user 123456789
[ничего больше]
```

### Гипотезы

#### Гипотеза А: Хендлер не зарегистрирован
**Проверка:** Добавил принт в начале start.py  
**Результат:** ❌ Принт не сработал → хендлер не вызывается

#### Гипотеза Б: Router не подключён к dispatcher
**Проверка:** Проверил app/bots/dispatcher.py  
**Результат:** ✅ Нашёл проблему!

В dispatcher.py:
```python
from app.bots.handlers import start  # Импортирован
# Но не зарегистрирован!
# dp.include_router(start.router) ← отсутствует
```

---

### 🔍 Корневая причина
Router из start.py не добавлен в dispatcher через include_router()

### ✅ Исправление
```diff
  from app.bots.handlers import start, admin
  
  dp = Dispatcher()
  dp.include_router(admin.router)
+ dp.include_router(start.router)
```

### 🧪 Проверка
- [x] /start отвечает меню
- [x] В логах: "Handler start called"
- [x] Другие команды работают

---

**Проблема решена. Продолжить задачу?**
```

## Инструменты отладки

### Логирование
```python
from app.utils.logger import get_logger
logger = get_logger(__name__)

logger.debug("Переменные: x=%s, y=%s", x, y)
logger.info("Вызов функции parse_product")
logger.error("Ошибка парсинга: %s", exc_info=True)
```

### Playwright для веб-отладки
```
1. Открыть web.telegram.org
2. Отправить команду боту
3. Сделать скриншот как доказательство
4. Проверить network requests если нужно
```

### Python debugger
```python
import pdb; pdb.set_trace()  # Точка останова
# В отладчике:
# n (next), s (step), c (continue)
# p variable (print)
# l (list code)
```

## Правила отладки

1. **Полный текст ошибки** — никогда не обрезать
2. **Одна гипотеза за раз** — проверять последовательно
3. **Минимальный сценарий** — убрать лишнее
4. **Записывать результат** — что проверил, что нашёл
5. **Проверка после исправления** — тест + регрессия
