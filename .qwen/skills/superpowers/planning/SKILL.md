# Суперспособность: Planning (Создание планов)

## Назначение
Создание детального пошагового плана реализации задачи после brainstorming.

## Когда использовать
- После выбора варианта решения (brainstorming)
- Перед началом реализации многошаговой задачи
- Когда задача требует 3+ шагов
- Для сложных изменений в нескольких файлах

## Алгоритм planning

### Шаг 1: Декомпозиция
Разбить задачу на атомарные шаги:
- Один шаг = одно действие (прочитать, изменить, создать, тестировать)
- Шаг должен быть выполним за 5-15 минут
- Минимум зависимостей между шагами

### Шаг 2: Оценка рисков
Для каждого шага определить:
- ⚠️ Что может пойти не так
- 🔄 План Б если шаг не получится
- 📦 Зависимости (нужно ли что-то заранее)

### Шаг 3: Приоритизация
Порядок выполнения:
1. Подготовительные шаги (создать файлы, импорты)
2. Основные изменения (логика)
3. Тесты и проверка
4. Документация

### Шаг 4: Валидация плана
Проверить план:
- [ ] Все шаги конкретные и измеримые
- [ ] Есть тесты после изменений кода
- [ ] Есть откат если что-то пойдёт не так
- [ ] Пользователь понял и согласен

## Шаблон плана

```markdown
# План: [Название задачи]

## 🎯 Цель
[Одно предложение что получится в результате]

## 📋 Шаги реализации

### Этап 1: Подготовка
| # | Шаг | Файлы | Риск |
|---|-----|-------|------|
| 1.1 | [Действие] | `путь` | Низкий |
| 1.2 | [Действие] | `путь` | Средний |

### Этап 2: Реализация
| # | Шаг | Файлы | Риск |
|---|-----|-------|------|
| 2.1 | [Действие] | `путь` | Средний |

### Этап 3: Тесты
| # | Шаг | Файлы | Риск |
|---|-----|-------|------|
| 3.1 | [Действие] | `путь` | Низкий |

## ⚠️ Риски и план Б

### Риск 1: [Описание]
**План Б:** [Альтернатива]

### Риск 2: [Описание]
**План Б:** [Альтернатива]

## ✅ Критерии завершения
- [ ] Критерий 1
- [ ] Критерий 2
- [ ] Критерий 3

---

**Готов начать Этап 1? Подтверди и я выполню Шаг 1.1**
```

## Пример плана

```markdown
# План: Парсинг Myntra

## 🎯 Цель
Добавить поддержку Myntra.in в selenium_service.py с AI экстракцией

## 📋 Шаги реализации

### Этап 1: Подготовка
| # | Шаг | Файлы | Риск |
|---|-----|-------|------|
| 1.1 | Прочитать документацию Myntra | Context7 | Низкий |
| 1.2 | Изучить структуру HTML Myntra | Playwright | Низкий |
| 1.3 | Создать parsers/myntra.py | Новый файл | Низкий |

### Этап 2: Реализация
| # | Шаг | Файлы | Риск |
|---|-----|-------|------|
| 2.1 | Реализовать extract_title() | myntra.py | Средний |
| 2.2 | Реализовать extract_price() | myntra.py | Средний |
| 2.3 | Реализовать extract_images() | myntra.py | Высокий |
| 2.4 | Интеграция с OpenRouter | myntra.py | Средний |

### Этап 3: Тесты
| # | Шаг | Файлы | Риск |
|---|-----|-------|------|
| 3.1 | Написать unit тесты | test_myntra.py | Низкий |
| 3.2 | Интеграционный тест | Playwright | Средний |
| 3.3 | Тест в Telegram | Ручной | Низкий |

## ⚠️ Риски и план Б

### Риск 1: Myntra блокирует Selenium
**План Б:** Использовать proxy + увеличить задержки

### Риск 2: Lazy loading изображений
**План Б:** Скроллить страницу перед скриншотом

### Риск 3: Другая структура цен
**План Б:** Добавить fallback парсинг через AI

## ✅ Критерии завершения
- [ ] Парсит название, цену, 5+ изображений
- [ ] AI экстракция работает
- [ ] Тесты проходят
- [ ] Протестировано в Telegram

---

**Готов начать Этап 1?**
```

## Правила планирования

1. **Один шаг за раз** — не выполнять следующий без подтверждения
2. **Тест после изменений** — каждый этап кода → тест
3. **Минимум 3 шага** — если меньше, задача слишком простая
4. **Максимум 10 шагов** — если больше, разбить на подзадачи
5. **Файлы явно** — указывать какие файлы меняются

## Связанные навыки
- `superpowers/brainstorming` — перед планированием
- `superpowers/executing` — выполнение плана
- `superpowers/verification` — проверка каждого шага
