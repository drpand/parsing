# Суперспособность: Executing (Выполнение планов)

## Назначение
Систематическое выполнение плана шаг за шагом с проверкой после каждого шага.

## Когда использовать
- Есть утверждённый план (planning)
- Реализация многошаговой задачи
- Треббуется дисциплина выполнения

## Алгоритм executing

### Перед началом
1. Прочитать план полностью
2. Убедиться что понятен каждый шаг
3. Подготовить окружение (открыть файлы, запустить тесты)
4. Получить подтверждение пользователя

### Выполнение шага
```
1. Прочитать текущий шаг плана
2. Выполнить ровно то что указано (не больше)
3. Проверить результат (тест, лог, скриншот)
4. Сообщить о завершении шага
5. Ждать подтверждения следующего шага
```

### Если шаг не получился
```
Попытка 1:
- Проанализировать ошибку
- Исправить
- Попробовать снова

Попытка 2:
- Если та же ошибка → СТОП
- Объяснить проблему пользователю
- Предложить альтернативу или план Б
```

### После шага
1. Обновить прогресс: "Шаг 2.1 из 10 ✅"
2. Кратко описать что сделано
3. Показать результат (код, тест, скриншот)
4. Спросить продолжения

## Шаблон отчёта о шаге

```markdown
### Шаг X.Y ✅

**Выполнено:** [Что сделал]

**Результат:**
```[код/лог/вывод]```

**Проверка:**
- [ ] Тест прошёл
- [ ] Ошибок нет
- [ ] Соответствует плану

---

**Продолжить Шаг X.Z?**
```

## Пример выполнения

```markdown
### Шаг 1.3 ✅

**Выполнено:** Создан файл app/services/parsers/myntra.py

**Результат:**
```python
# app/services/parsers/myntra.py
from .base import BaseParser

class MyntraParser(BaseParser):
    MARKETPLACE = 'myntra.in'
    
    async def parse_product(self, url: str) -> Product:
        # Загрузка страницы
        await self.page.goto(url)
        await self.page.wait_for_load_state('networkidle')
        
        # Скриншот для AI
        screenshot = await self.page.screenshot()
        
        # AI экстракция
        return await self.extract_with_ai(screenshot)
```

**Проверка:**
- [x] Файл создан
- [x] Синтаксис верный
- [x] Наследует BaseParser

---

**Продолжить Шаг 2.1 (extract_title)?**
```

## Правила выполнения

1. **Точно по плану** — не импровизировать без необходимости
2. **Один файл за раз** — не менять несколько файлов в одном шаге
3. **Тест сразу** — написал код → запустил тест
4. **Честность об ошибках** — не скрывать проблемы
5. **Пауза на подтверждение** — ждать "да" перед следующим шагом

## Обработка ошибок

### Ошибка в коде
```
1. Показать полный текст ошибки (не обрезать)
2. Объяснить своими словами что пошло не так
3. Предложить исправление
4. Исправить → тестировать снова
```

### Ошибка в плане
```
1. Признать что план не работает
2. Объяснить почему
3. Предложить скорректировать план
4. Ждать подтверждения изменений
```

### Таймаут / Блокировка
```
1. Показать что заблокировано (URL, сервис)
2. Предложить альтернативу (proxy, другой подход)
3. Спросить токены/ключи если нужны
```
