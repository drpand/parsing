# Суперспособность: Brainstorming (Генерация идей)

## Назначение
Исследование проблемы с разных сторон, генерация вариантов решений перед началом реализации.

## Когда использовать
- Получил сложную задачу без очевидного решения
- Нужно выбрать между несколькими подходами
- Перед началом большой фичи
- Застрял — ни один вариант не работает

## Алгоритм brainstorming

### Шаг 1: Понимание проблемы
```
1. Переформулировать задачу своими словами
2. Выделить ключевые требования
3. Определить ограничения (время, ресурсы, зависимости)
4. Задать уточняющие вопросы если нужно
```

### Шаг 2: Исследование контекста
```
1. Изучить существующий код (похожие решения?)
2. Проверить документацию библиотек (Context7)
3. Найти ограничения архитектуры
4. Оценить риски
```

### Шаг 3: Генерация вариантов
```
Минимум 3 подхода:
- Вариант А: Консервативный (минимум изменений)
- Вариант Б: Сбалансированный (оптимальный)
- Вариант В: Радикальный (полная переделка)
```

### Шаг 4: Анализ вариантов
Для каждого варианта оценить:
- ✅ Преимущества
- ❌ Недостатки
- ⏱ Время реализации
- 🔒 Риски
- 📦 Зависимости

### Шаг 5: Рекомендация
```
1. Выбрать лучший вариант
2. Объяснить почему
3. Предложить план реализации
4. Ждать подтверждения пользователя
```

## Пример использования

**Задача:** "Добавить парсинг Myntra"

**Brainstorming:**
```
## 🧠 Brainstorming: Парсинг Myntra

### Понимание задачи
Нужно расширить selenium_service.py поддержкой Myntra.in
Текущий код заточен под Zara, структура HTML другая

### Требования
- Извлекать: название, цена, изображения, описание
- Поддержка категорий: men, women, shoes
- AI экстракция через OpenRouter

### Ограничения
- Myntra использует lazy loading изображений
- Антибот защита (но слабее чем Zara)
- Цены в ₹ (уже поддерживается)

---

### Варианты решений

#### Вариант А: Адаптировать существующий парсер
**Описание:** Добавить условия if marketplace == 'myntra' в текущий код

✅ **Плюсы:**
- Минимум нового кода
- Общая логика кэширования
- Единый интерфейс

❌ **Минусы:**
- Много условий if/else
- Сложнее тестировать
- Риск сломать Zara

⏱ **Время:** ~2 часа

---

#### Вариант Б: Отдельный класс MyntraParser ← Рекомендуемый
**Описание:** Создать app/services/parsers/myntra.py с наследованием

✅ **Плюсы:**
- Чистая архитектура
- Легко тестировать отдельно
- Не затронет Zara
- Масштабируемо (другие маркетплейсы)

❌ **Минусы:**
- Больше файлов
- Дублирование общей логики

⏱ **Время:** ~3 часа

---

### 🎯 Рекомендация

**Выбираем Вариант Б** потому что:
1. Чистая архитектура — каждый парсер в своём файле
2. Безопасно — Zara не сломается
3. Масштабируемо — легко добавить Ajio, Nykaa

### План реализации
1. Создать app/services/parsers/myntra.py
2. Реализовать extract_price(), extract_images()
3. Написать тесты
4. Интегрировать в admin_parse.py

**Готов начать?**
```
