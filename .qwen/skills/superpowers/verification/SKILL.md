# Суперспособность: Verification Before Completion (Проверка перед завершением)

## Назначение
Обязательная проверка что код реально работает, а не "нет ошибок в коде".

## Когда использовать
- Перед пометкой задачи как завершённой
- После написания кода
- Перед коммитом в git
- Когда пользователь спрашивает "готово?"

## Принцип Source of Truth

```
❌ Ложный успех: "Код без синтаксических ошибок"
✅ Истинный успех: "Реальный тест проходит"
```

## Алгоритм verification

### Уровень 1: Синтаксис
```
1. Проверить синтаксис (Python: python -m py_compile)
2. Проверить импорты (нет ImportError)
3. Проверить типизацию (mypy, если используется)
```

### Уровень 2: Unit тесты
```
1. Запустить тесты на изменённый модуль
2. Все тесты должны пройти
3. Покрыть новые функции тестами
```

### Уровень 3: Integration тесты
```
1. Проверить взаимодействие модулей
2. Проверить базу данных
3. Проверить внешние API
```

### Уровень 4: E2E тест (реальный сценарий)
```
1. Открыть Telegram / web.telegram.org
2. Выполнить сценарий пользователя
3. Сделать скриншот как доказательство
4. Проверить что результат соответствует ожиданию
```

## Чеклист верификации

### Для нового кода
- [ ] Синтаксис без ошибок
- [ ] Unit тесты написаны и проходят
- [ ] Интеграция с другими модулями работает
- [ ] E2E тест в Telegram пройден
- [ ] Логи чистые (нет warning/error)

### Для исправления бага
- [ ] Тест на баг написан
- [ ] Тест проходит с исправлением
- [ ] Регрессия: другие тесты не сломаны
- [ ] Пользователь подтвердил что работает

### Для рефакторинга
- [ ] Все существующие тесты проходят
- [ ] Производительность не ухудшилась
- [ ] Код стал понятнее
- [ ] Нет нового дублирования

## Инструменты проверки

### Python
```bash
# Синтаксис
python -m py_compile file.py

# Тесты
pytest tests/ -v --cov=app

# Типы
mypy app/

# Линтер
ruff check app/
```

### Playwright (Telegram E2E)
```
1. mcp__playwright__browser_navigate → web.telegram.org
2. mcp__playwright__browser_click → кнопки
3. mcp__playwright__browser_type → сообщения
4. mcp__playwright__browser_take_screenshot → доказательство
```

### Логи
```bash
# Последние 50 строк
Get-Content bot.log -Tail 50 -Wait

# Поиск ошибок
Select-String -Path bot.log -Pattern "ERROR"
```

## Правила верификации

1. **E2E обязателен** — для бота тест в Telegram
2. **Скриншот как доказательство** — через Playwright
3. **Полные логи ошибок** — не скрывать проблемы
4. **Регрессия** — проверить что не сломало другое
5. **Подтверждение пользователя** — спросить "Продолжить?"
