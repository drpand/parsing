# Навык: Паттерны разработки (Patterns Memory)

## Назначение
Ведение файла `.qwen/memory/patterns.md` для сохранения рабочих паттернов и антипаттернов.

## Когда использовать
- После успешного решения сложной задачи — сохранить паттерн
- После ошибки которую можно избежать — добавить антипаттерн
- Перед началом похожей задачи — проверить существующие паттерны

## Формат patterns.md

```markdown
# Паттерны проекта

## 📁 Архитектура

### Паттерн: Название
**Когда:** Ситуация где применяется  
**Проблема:** Что решает  
**Решение:** Как реализовать  
**Пример:** Код или псевдокод

---

## 🐛 Антипаттерны

### Антипаттерн: Название
**Проблема:** Что идёт не так  
**Как избежать:** Правильный подход  
**Пример:** Плохой код → Хороший код
```

## Категории паттернов

1. **Архитектура** — структура проекта, модули
2. **Безопасность** — работа с секретами, токенами
3. **Тестирование** — подходы к тестам
4. **Отладка** — debugging workflows
5. **Производительность** — оптимизации
6. **Git** — workflow, ветвление

## Примеры паттернов

### Паттерн: Source of Truth
**Когда:** Проверка что код работает  
**Проблема:** "Нет ошибок" ≠ "Работает"  
**Решение:** Всегда проверять реальным тестом  
**Пример:**
```
❌ Успех: код без синтаксических ошибок
✅ Успех: бот ответил в Telegram правильно
```

### Паттерн: Один файл за раз
**Когда:** Рефакторинг нескольких файлов  
**Проблема:** Изменение 5 файлов сразу → сложно откатить  
**Решение:** Менять один файл → тестировать → следующий  
**Пример:**
```
1. Прочитать файл А
2. Изменить файл А
3. Тестировать
4. Перейти к файлу Б
```

### Паттерн: Context7 перед кодом
**Когда:** Использование библиотеки  
**Проблема:** API меняется, память устаревает  
**Решение:** Сначала query-docs, потом код  
**Пример:**
```
1. resolve-library-id: "aiogram"
2. query-docs: "how to create router"
3. Писать код по документации
```

## Примеры антипаттернов

### Антипаттерн: Галлюцинация токенов
**Проблема:** Придумывание API ключей вместо запроса  
**Как избежать:** Всегда спрашивать у пользователя  
**Пример:**
```
❌ "Возьму токен из примера"
✅ "Нужен BOT_TOKEN. Запросить у пользователя"
```

### Антипаттерн: Третий раз тот же
**Проблема:** Повторение неудачного решения 3+ раз  
**Как избежать:** Стоп после 2 попыток, анализ  
**Пример:**
```
Попытка 1: ошибка → исправить
Попытка 2: ошибка → СТОП
Действия: Объяснить проблему, предложить альтернативу
```

## Алгоритм добавления паттерна

1. **Идентифицировать** — ситуация повторяется?
2. **Сформулировать** — проблема → решение
3. **Пример** — код до/после или псевдокод
4. **Категория** — куда добавить (архитектура, тесты...)
5. **Записать** — в patterns.md
