# Навык: Управление задачами (Task Memory)

## Назначение
Ведение и актуализация файла задач `.qwen/memory/tasks.md` для отслеживания прогресса разработки.

## Когда использовать
- В начале сессии — прочитать текущие задачи
- При получении новой задачи — добавить в список
- После завершения шага — обновить статус
- В конце сессии — зафиксировать прогресс

## Формат tasks.md

```markdown
# Задачи проекта

## Слои (Layers)

### 🔴 Текущий слой
- [ ] Конкретная задача 1
- [x] Завершённая задача

### 🟡 Следующий слой
- [ ] Задача на потом

### ⚪ Будущие слои
- [ ] Идеи на будущее
```

## Правила

1. **Одна задача за раз** — в работе только одна задача из текущего слоя
2. **Статусы**:
   - `[ ]` — pending (ожидает)
   - `[~]` — in_progress (в работе)
   - `[x]` — completed (завершено)
3. **Декомпозиция** — большие задачи разбивать на подзадачи
4. **Контекст** — каждая задача должна быть самодостаточной

## Алгоритм работы

### Начало сессии
1. Прочитать `.qwen/memory/tasks.md`
2. Определить текущую задачу (первая `[ ]` в текущем слое)
3. Сообщить пользователю: "Продолжаем с: [задача]"

### Во время работы
1. Перед началом задачи → обновить статус на `[~]`
2. После завершения → обновить на `[x]`
3. Следующую задачу отметить `[~]`

### Конец сессии
1. Обновить `.qwen/memory/sessions.md` с итогами
2. Проверить актуальность layers
3. При необходимости переместить задачи между слоями

## Примеры

### Добавить задачу
```markdown
### 🔴 Текущий слой
- [x] Настроить подключение к базе данных
- [~] Создать модель User
- [ ] Реализовать CRUD операции
```

### Декомпозиция
```markdown
- [ ] Реализовать парсинг
  - [ ] Selenium: загрузка страницы
  - [ ] Extract: CSS селекторы
  - [ ] OpenRouter: AI экстракция
  - [ ] Валидация данных
```
