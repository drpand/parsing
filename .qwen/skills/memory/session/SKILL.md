# Навык: Журнал сессий (Session Memory)

## Назначение
Ведение файла `.qwen/memory/sessions.md` для документирования каждой сессии разработки.

## Когда использовать
- В начале сессии — прочитать последнюю запись для контекста
- В конце сессии — записать результаты, проблемы, решения

## Формат sessions.md

```markdown
# Журнал сессий

## Сессия #N — YYYY-MM-DD

### Контекст
- Продолжение с: [задача из tasks.md]
- Цель сессии: что планировалось

### Что сделано
- ✅ Шаг 1
- ✅ Шаг 2

### Проблемы и решения
| Проблема | Решение | Статус |
|----------|---------|--------|
| Описание | Как решил | ✅ Решено |

### Изменения в коде
- `путь/к/файлу.py` — описание изменений

### Заметки
- Важные инсайты
- Что сработало хорошо
- Что улучшить в следующий раз

### Следующая сессия
- [ ] Задача 1
- [ ] Задача 2
```

## Правила

1. **Хронология** — сессии нумеруются по порядку
2. **Конкретика** — описывать что именно сделано, не "работал над кодом"
3. **Проблемы** — фиксировать ошибки и способы решения
4. **Связь с tasks.md** — указывать какие задачи закрыты

## Алгоритм

### Начало сессии
1. Прочитать последнюю сессию
2. Извлечь контекст: что делали, какие проблемы остались
3. Сообщить: "Продолжаем сессию #N, последняя задача: [задача]"

### Конец сессии
1. Создать новую запись (или обновить текущую)
2. Заполнить секции:
   - Что сделано (список ✅)
   - Проблемы (таблица)
   - Изменения (файлы)
   - Следующие шаги
3. Обновить номер последней сессии

## Пример записи

```markdown
## Сессия #42 — 2026-03-23

### Контекст
- Продолжение с: Реализация парсинга Zara
- Цель: Завершить selenium_service.py, протестировать

### Что сделано
- ✅ Добавлен stealth режим для ChromeDriver
- ✅ Реализован extract_price() для ₹ и Rs.
- ✅ Написан тест test_parse_zara_product()
- ✅ Протестировано через Playwright — работает

### Проблемы и решения
| Проблема | Решение | Статус |
|----------|---------|--------|
| Zara блокирует Selenium | Добавлен webdriver-mask + user-agent рандом | ✅ Решено |
| Цена не извлекается | Добавлен fallback формат Rs. | ✅ Решено |

### Изменения в коде
- `app/services/selenium_service.py` — +200 строк, stealth
- `tests/test_parser.py` — +50 строк, тесты

### Заметки
- Stealth работает с 3 задержками: 1s, 2s, random(1-3s)
- Playwright тест обязателен перед коммитом

### Следующая сессия
- [ ] Парсинг Myntra (другая структура)
- [ ] Кэширование AI ответов
```
